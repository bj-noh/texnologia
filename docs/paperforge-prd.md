# PaperForge 제품 요구사항 문서

## 1. 제품 개요

PaperForge는 macOS용 독립 LaTeX writing IDE이다. 논문, 기술 문서, 연구 보고서를 빠르게 작성하고 컴파일 오류를 즉시 이해하며 수정할 수 있도록 설계한다. 핵심 차별점은 "빠른 논문 작성"과 "오류 수정 흐름"에 최적화된 macOS-native 경험이다.

PaperForge는 Texifier, TeXShop, VS Code LaTeX Workshop이 포괄하는 기능 범주를 참고하되, 브랜드, UI 구조, 고유 기능 구현은 복제하지 않는다. 목표는 범용 코드 에디터가 아니라 연구자가 논문을 끝까지 완성하는 데 필요한 집중형 LaTeX 작업 공간이다.

## 2. 목표 사용자

### 주요 페르소나

- 논문 작성자: 학회/저널 제출 형식에 맞춰 빠르게 초안, 수정본, 최종본을 작성한다.
- 대학원생: LaTeX 숙련도가 낮거나 중간 수준이며, 오류 메시지 해석과 템플릿 설정에서 자주 막힌다.
- 교수: 여러 논문 프로젝트를 관리하고, 학생 원고를 리뷰하며, 변경 사항을 빠르게 확인해야 한다.
- 연구자: 수식, 그림, 참고문헌, 다중 파일 구조를 안정적으로 다루어야 한다.
- 엔지니어: 기술 보고서, 스펙, 백서, RFC 스타일 문서를 LaTeX로 작성한다.

## 3. 제품 목표

- LaTeX 초안 작성부터 PDF 확인까지의 반복 시간을 줄인다.
- 컴파일 오류를 사용자가 이해할 수 있는 행동 가능한 수정 단위로 변환한다.
- macOS 앱다운 빠른 실행, 안정적인 파일 처리, 네이티브 단축키, 미려한 PDF 미리보기를 제공한다.
- 논문 프로젝트의 구조, 참고문헌, 그림, 로그, 출력물을 한 공간에서 관리한다.
- 초보자는 덜 막히고, 숙련자는 더 빠르게 작성할 수 있는 IDE를 만든다.

## 4. 핵심 가치 제안

- 오류 중심 워크플로우: 원본 로그가 아니라 원인, 위치, 추천 수정, 관련 파일을 우선 보여준다.
- 논문 작성 최적화: 섹션, 수식, 그림, 표, 참고문헌, cross-reference 작성 흐름을 단축한다.
- macOS-native UX: Command Palette, Spotlight 스타일 프로젝트 검색, Quick Look, 탭/윈도우, 다크 모드, 키체인/파일 권한 모델을 자연스럽게 지원한다.
- 빠른 PDF 피드백: 저장 후 자동 빌드, 증분 빌드, forward/inverse search, 변경 위치 중심 미리보기를 제공한다.
- 독립 앱: VS Code 확장 의존 없이 설치 후 바로 논문 작성이 가능해야 한다.

## 5. 기능 로드맵

### MVP

- 프로젝트 열기: `.tex`, `.bib`, 이미지, 스타일 파일을 포함한 폴더 기반 프로젝트 지원
- LaTeX 에디터: 문법 강조, 자동 들여쓰기, 괄호/환경 자동 완성, 기본 스니펫
- PDF 미리보기: 에디터와 PDF 분할 보기, 자동 갱신, 페이지/확대/검색
- 빌드 시스템: `latexmk` 기반 기본 빌드, TeX Live/MacTeX 감지, 빌드 명령 설정
- 오류 패널: 파일, 줄 번호, 오류 유형, 원문 로그, 간단한 설명 표시
- Forward search: 소스 위치에서 PDF 위치로 이동
- 프로젝트 아웃라인: `section`, `subsection`, `label`, `figure`, `table` 탐색
- 참고문헌 기본 지원: `.bib` 파일 열기, citation key 자동완성
- macOS 기본 기능: 다크 모드, 자동 저장 옵션, 최근 프로젝트, 표준 단축키

### Beta

- 오류 해석 고도화: 흔한 LaTeX 오류에 대한 원인 설명과 수정 제안
- Inverse search: PDF 클릭 위치에서 소스 파일/줄로 이동
- 다중 파일 프로젝트 인식: `\input`, `\include`, `\bibliography`, `\addbibresource` 그래프 표시
- 논문 작성 도우미: figure/table/equation/algorithm 삽입 패널
- Cross-reference 자동완성: `\ref`, `\cref`, `\autoref`, `\eqref` 후보 제공
- 빌드 프로파일: pdfLaTeX, XeLaTeX, LuaLaTeX, BibTeX, Biber 프리셋
- 템플릿 시작 화면: article, thesis, conference paper, technical report
- Git 변경 표시: 수정 라인 표시, 변경 파일 목록, 외부 Git 도구 연동

### v1.0

- Paper Diagnostics: 오류, 경고, overfull box, missing reference, citation 문제를 통합 점검
- Quick Fix: 누락 패키지, 닫히지 않은 환경, 잘못된 citation/reference에 대한 원클릭 수정
- Symbol/command palette: LaTeX 명령, 수학 기호, 프로젝트 파일, 빌드 액션 검색
- Bibliography manager: citation 검색, BibTeX/BibLaTeX 필드 검증, 중복 key 감지
- Review mode: PDF 주석, TODO, source comment를 한 목록에서 관리
- 제출 준비 체크리스트: auxiliary 파일 정리, 이미지 누락 확인, bibliography 포함 여부 확인
- 성능 목표 달성: 중형 논문 프로젝트 기준 저장 후 미리보기 갱신 2초 이내를 목표로 최적화

### v2.0

- AI-assisted repair: 컴파일 로그와 주변 소스를 기반으로 설명/수정 diff 제안
- 논문 구조 품질 점검: 섹션 길이, 미참조 그림/표, 정의되지 않은 약어, 반복 표현 탐지
- 협업 워크플로우: 변경 제안, 코멘트 스레드, 리뷰 상태 추적
- 학회/저널 제출팩 생성: 클래스/스타일/그림/bib/output 정리 및 압축
- 클라우드 동기화 선택 지원: iCloud Drive 프로젝트 안정성 강화, 충돌 감지
- 플러그인/확장 API: 빌드 도구, 템플릿, 진단 규칙 확장

## 6. MVP 사용자 스토리

- 대학원생으로서, 기존 LaTeX 프로젝트 폴더를 열고 별도 설정 없이 PDF를 빌드하고 싶다.
- 논문 작성자로서, 저장할 때마다 PDF가 자동 갱신되어 문서 결과를 즉시 확인하고 싶다.
- LaTeX 초보자로서, 컴파일 실패 시 긴 로그 대신 어느 파일의 몇 번째 줄에서 무엇을 고쳐야 하는지 보고 싶다.
- 연구자로서, `\cite{}` 입력 중 `.bib`의 citation key 후보를 빠르게 선택하고 싶다.
- 교수로서, PDF의 특정 문단을 보다가 해당 소스 위치로 바로 이동하고 싶다.
- 엔지니어로서, 여러 `.tex` 파일로 나뉜 기술 문서를 프로젝트 아웃라인에서 탐색하고 싶다.
- 숙련 사용자로서, XeLaTeX 또는 LuaLaTeX 빌드 설정을 프로젝트별로 저장하고 싶다.

## 9. 주요 요구사항

### 편집

- 대용량 `.tex` 파일에서도 입력 지연이 없어야 한다.
- `\begin{}`/`\end{}`, `$...$`, `\[...\]`, `{}` 쌍을 안정적으로 보조해야 한다.
- citation, reference, label, file path 자동완성을 제공해야 한다.
- 여러 파일에 분산된 문서 구조를 사용자가 명시적으로 추적하지 않아도 탐색 가능해야 한다.

### 빌드

- MacTeX/TeX Live 설치 여부와 실행 경로를 자동 감지해야 한다.
- 기본 빌드는 `latexmk`를 사용하되, 사용자가 엔진과 옵션을 프로젝트별로 바꿀 수 있어야 한다.
- 빌드 실패 시 PDF 미리보기, 로그, 오류 목록이 서로 연결되어야 한다.
- 빌드 산출물과 auxiliary 파일을 사용자가 쉽게 정리할 수 있어야 한다.

### 오류 수정

- 오류는 "문제 위치", "원인 설명", "추천 행동", "원문 로그" 순서로 표시한다.
- 한 오류가 여러 로그 라인을 생성하더라도 사용자는 하나의 문제로 이해할 수 있어야 한다.
- 경고와 오류를 구분하고, overfull/underfull box는 별도 품질 진단으로 분류한다.

### PDF 미리보기

- PDF는 저장/빌드 후 자동 갱신되어야 한다.
- 소스와 PDF 간 위치 이동을 지원해야 한다.
- 검색, 확대/축소, 페이지 이동, 탭 분리 보기를 제공해야 한다.

## 10. 성공 지표

- 신규 사용자가 샘플 논문 프로젝트를 열고 첫 PDF 빌드에 성공하는 시간: 3분 이내
- 컴파일 오류 발생 후 사용자가 문제 위치에 도달하는 시간: 10초 이내
- 중형 프로젝트 저장 후 PDF 갱신 체감 시간: 2초 이내 목표
- Beta 사용자 중 주간 활성 사용자 비율: 40% 이상
- 오류 패널에서 제공한 위치가 실제 수정 위치와 일치하는 비율: 80% 이상

## 11. 리스크와 가정

### 리스크

- LaTeX 로그 형식이 패키지와 엔진별로 다양해 오류 원인 추론 정확도가 낮아질 수 있다.
- SyncTeX 기반 forward/inverse search는 프로젝트 구조와 빌드 설정에 따라 안정성이 달라질 수 있다.
- macOS sandbox, 파일 권한, iCloud Drive 동기화가 프로젝트 파일 접근에 영향을 줄 수 있다.
- TeX Live/MacTeX 미설치 사용자에게 온보딩 마찰이 발생할 수 있다.
- AI-assisted repair는 잘못된 수정 제안을 만들 수 있으므로 자동 적용에는 신중한 UX가 필요하다.
- 대형 논문 프로젝트에서 실시간 진단과 PDF 갱신이 성능 병목을 만들 수 있다.

### 가정

- 초기 타깃 사용자는 macOS에서 이미 LaTeX 또는 MacTeX를 사용하거나 설치할 의향이 있다.
- 사용자는 범용 IDE보다 논문 작성에 특화된 빠른 흐름에 가치를 느낀다.
- MVP에서는 로컬 파일 기반 단일 사용자 워크플로우가 가장 중요하다.
- 오류 수정 경험이 PaperForge의 핵심 차별화와 유료 전환 요인이 된다.
- v1.0 이전에는 완전한 협업 기능보다 안정적인 빌드/미리보기/진단이 더 중요하다.

## 12. 출시 기준

### MVP 완료 기준

- 대표 샘플 프로젝트 5종에서 열기, 편집, 빌드, PDF 미리보기가 동작한다.
- 주요 오류 20종 이상이 오류 패널에 위치와 원문 로그로 표시된다.
- citation/reference/label 자동완성이 기본 프로젝트에서 동작한다.
- 앱 종료/재시작 후 최근 프로젝트와 빌드 설정이 보존된다.

### Beta 완료 기준

- 실사용 논문 프로젝트 30개 이상에서 빌드 성공/실패 상태를 정확히 표시한다.
- 흔한 오류 50종 이상에 대해 설명 또는 수정 힌트를 제공한다.
- forward/inverse search가 SyncTeX 지원 프로젝트에서 안정적으로 동작한다.
- 사용자가 템플릿에서 새 논문 프로젝트를 생성할 수 있다.

### v1.0 완료 기준

- Paper Diagnostics와 Quick Fix가 핵심 오류/경고 흐름에 통합된다.
- bibliography, cross-reference, 제출 전 점검 기능이 논문 작성 루프 안에서 자연스럽게 연결된다.
- 대형 프로젝트에서도 편집 입력 지연과 PDF 갱신 지연이 허용 범위 안에 있다.
- 충돌, 빌드 실패, 경로 오류, TeX 미설치 상태에 대한 복구 UX가 명확하다.
